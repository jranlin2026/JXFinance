# 订单明细融合表规则

## 一句话结论

订单明细融合表是利润统计的订单行级底表：以订单表为主表，按每一行订单输出，并挂接资金流水的结算到账金额、运费险费用和商品单价表成本。

## 适用范围

用于抖音店铺自动化结算中，给财务和开发提供可追溯的订单明细底表。它不是达人汇总表，而是后续利润计算和异常复核的明细依据。

本规则已用 `风华服饰时尚小铺` 2026 年 5 月订单表、资金流水表、运费险表、商品单价表验证通过。

## 输入表

```text
订单表：主表，每一行订单都输出到明细表。
资金流水表：提供结算到账金额。
运费险表：可选，提供运费险费用；缺失时按 0 费用处理。
商品单价表：可选，提供产品单件成本；缺失时按 0 成本处理。
```

## 输出字段

字段顺序固定如下：

```text
订单月份
主订单编号
子订单编号
结算到账金额
运费险
商品数量
商家编码
商品单价
订单应付金额
产品单件成本
产品总成本
订单提交时间
订单完成时间
支付完成时间
达人ID
达人昵称
发货时间
```

## 字段口径

### 订单月份

```text
订单月份 = 订单提交时间的月份
格式：YYYY-MM
```

### 订单基础字段

以下字段直接来自订单表：

```text
主订单编号
子订单编号
商品数量
商家编码
商品单价
订单应付金额
订单提交时间
订单完成时间
支付完成时间
达人ID
达人昵称
发货时间
```

达人昵称为空时：

```text
达人昵称 = 商品卡流量
达人ID = 空
```

### 结算到账金额

结算到账金额来自资金流水表。

计算方式：

```text
结算到账金额 = 资金流水入账金额 - 资金流水出账金额
```

方向规则：

```text
动账方向 = 入账 -> 动账金额记为正数
动账方向 = 出账 -> 动账金额记为负数
```

挂接规则：

```text
1. 优先用资金流水表的 子订单号 匹配订单表的 子订单编号。
2. 如果资金流水表子订单号为空，再用 订单号 匹配订单表的 主订单编号。
3. 订单行匹配不到资金流水时，结算到账金额留空。
4. 正数和负数都保留，负数是正常情况。
```

### 运费险

运费险来自运费险表。

计算条件：

```text
保费状态 = 已扣减 -> 计入运费险
保费状态 = 不扣减 -> 不计入运费险
```

挂接规则：

```text
1. 使用运费险表的 订单编号 匹配订单表的 子订单编号。
2. 不使用主订单编号兜底匹配运费险。
3. 订单行匹配不到已扣减运费险时，运费险留空或按 0 费用处理。
```

注意：

```text
订单明细融合表是行级追溯表。运费险只按子订单编号匹配；不能按主订单编号匹配，否则同一主订单下多子订单会导致运费险金额偏高。
```

### 产品单件成本

产品单件成本来自商品单价表。

匹配规则：

```text
订单表.商家编码 = 商品单价表.商家编码
产品单件成本 = 商品单价表.产品单价
```

### 产品总成本

产品总成本只有在订单结算到账金额大于 5 元时才计算。

计算方式：

```text
如果 结算到账金额 > 5：
产品总成本 = 产品单件成本 * 商品数量

如果 结算到账金额 <= 5 或结算到账金额为空：
产品总成本 = 空
```

注意：

```text
没有统计金额的字段统一留空，不填 0。
例如：没有结算到账金额、没有运费险、或不满足成本计算条件时，对应金额字段留空。
```

## 使用提醒

```text
订单明细融合表用于追溯订单行，不等同于最终汇总利润表。
需要按达人、订单或费用类型汇总时，应基于对应汇总口径另行聚合，避免因为订单多行导致费用重复累计。
```
