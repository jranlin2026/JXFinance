# 快递包裹数计算规则

## 适用范围

用于抖音店铺达人结算中的快递包裹数与快递成本计算。

## 已确认口径

快递包裹数只从订单表计算。

订单表中只有 `发货时间` 非空的订单，才认为产生快递包裹。

同一个主订单可能包含多个子订单。如果同一个主订单下存在多个不同的发货时间，则代表分多个包裹发货。

## 去重规则

按以下字段组合去重：

```text
达人分类 + 主订单编号 + 发货时间
```

每一个唯一组合计为 1 个快递包裹。

## 达人分类规则

用于包裹数汇总时，达人分类按以下规则处理：

```text
如果 流量来源 = 小店自卖 -> 商品卡流量
如果 达人昵称为空 -> 商品卡流量
否则 -> 使用达人昵称
```

后续如果接入达人映射配置，则可以把达人昵称先映射为结算分类，再参与包裹数汇总。

## 示例

一个主订单号下面有 5 个子订单号：

```text
主订单编号 = M001
子订单号 = S001, S002, S003, S004, S005
```

其中：

```text
S001, S002, S003 发货时间 = 2026-05-11 09:00:00
S004, S005       发货时间 = 2026-05-12 15:30:00
```

则：

```text
快递包裹数 = 2
```

因为同一个主订单存在 2 个不同发货时间。

## 快递成本公式

```text
快递成本 = 快递包裹数 × 单件快递费
```
