# 结算输出表标准规则

## 一句话结论

结算输出表是抖音店铺自动化结算的标准交付文件：用户至少上传订单明细表和资金流水明细表，商品成本明细表和运费险明细表可选；系统最终生成一个 Excel 工作簿，里面固定包含 `订单明细融合表` 和 `达人结算汇总表` 两个工作表。

## 输入表要求

### 必填表

```text
1. 抖店导出的订单明细表
2. 抖店导出的资金流水明细表
```

没有这两张表时不能生成标准结算输出表。

### 可选表

```text
1. 商品成本明细表
2. 运费险明细表
```

可选表缺失时的处理：

```text
没有商品成本明细表 -> 产品单件成本、产品总成本、产品成本按 0 成本处理
没有运费险明细表 -> 运费险、运费险费用按 0 费用处理
```

如果用户提供了商品成本明细表或运费险明细表，则按已确认算法计算，不能再人工估算。

## 标准输出

最终只需要交付一个 Excel 工作簿，建议命名为：

```text
结算输出表.xlsx
```

工作簿内固定包含 2 个工作表：

```text
1. 订单明细融合表
2. 达人结算汇总表
```

`订单明细融合表` 是订单行级追溯底表；`达人结算汇总表` 是最终达人利润结算表。

## 订单明细融合表口径

订单明细融合表以订单明细表为主表，每一行订单都输出。

固定字段：

```text
订单月份
主订单编号
子订单编号
结算到账金额
运费险
商品数量
商家编码
商品单价
订单应付金额
产品单件成本
产品总成本
订单提交时间
订单完成时间
支付完成时间
达人ID
达人昵称
发货时间
```

核心规则：

```text
订单月份 = 订单提交时间的月份，格式 YYYY-MM
达人昵称为空 -> 商品卡流量
达人昵称为空时，达人ID 留空
结算到账金额 = 资金流水入账金额 - 资金流水出账金额
产品总成本只在该订单行结算到账金额 > 5 元时计算
```

资金流水匹配：

```text
1. 如果资金流水有子订单号，只能匹配订单表的子订单编号。
2. 只有资金流水子订单号为空时，才允许用资金流水订单号匹配订单表主订单编号。
3. 不允许用一个子订单流水覆盖同一主订单下的其他子订单。
4. 正数和负数都保留。
```

运费险匹配：

```text
1. 只统计运费险明细表中 保费状态 = 已扣减 的记录。
2. 运费险明细表的 订单编号 只匹配订单明细表的 子订单编号。
3. 不使用主订单编号兜底匹配运费险。
4. 匹配不到子订单编号的运费险记录不归属到订单行，作为未匹配异常记录。
```

这是今天修正过的关键口径：订单明细融合表里的运费险不能比对主订单编号，否则同一主订单下多子订单会导致运费险金额偏高。

商品成本匹配：

```text
订单明细表.商家编码 = 商品成本明细表.商家编码
产品单件成本 = 商品成本明细表.产品单价

如果 结算到账金额 > 5：
产品总成本 = 产品单件成本 * 商品数量

如果 结算到账金额 <= 5 或为空：
产品总成本 = 0 或空，最终汇总时按 0 处理
```

## 达人结算汇总表口径

达人结算汇总表按以下分组汇总：

```text
订单月份 + 达人昵称 + 达人ID
```

固定字段：

```text
订单月份
达人昵称
达人ID
实付订单金额
实付订单数
快递包裹数
快递费用
运费险费用
结算到账金额
产品成本
成本总额
毛利润
销售额毛利率
```

实付订单金额：

```text
只统计 支付完成时间 非空的订单行

实付订单金额 =
订单应付金额
+ 平台实际承担优惠金额
+ 商家实际承担优惠金额
+ 达人实际承担优惠金额
```

实付订单数：

```text
实付订单数 = 支付完成时间非空订单中的去重子订单编号数
```

快递包裹数：

```text
只统计发货时间非空的订单
按 订单月份 + 达人昵称 + 达人ID + 主订单编号 + 发货时间 去重
每个唯一组合算 1 个快递包裹
```

快递费用：

```text
快递费用 = 快递包裹数 * 2.4
```

运费险费用：

```text
有运费险明细表时：
只统计 保费状态 = 已扣减 的支付保费
运费险明细表.订单编号 只匹配 订单明细表.子订单编号
按匹配到的订单月份 + 达人昵称 + 达人ID 汇总

没有运费险明细表时：
运费险费用 = 0
```

结算到账金额：

```text
按资金流水实际入账/出账计算，不看订单状态
结算到账金额 = 入账金额 - 出账金额
正数、负数都保留
```

产品成本：

```text
有商品成本明细表时：
只认商品成本明细表的产品单价
产品成本 = 产品单价 * 商品数量
但只统计结算到账金额 > 5 元的订单行

没有商品成本明细表时：
产品成本 = 0
```

利润公式：

```text
成本总额 = 快递费用 + 运费险费用 + 产品成本
毛利润 = 结算到账金额 - 成本总额
销售额毛利率 = 毛利润 / 实付订单金额
```

如果 `实付订单金额 = 0`，销售额毛利率留空，避免除以 0。

## 使用提醒

```text
订单明细融合表负责追溯，达人结算汇总表负责最终结算。
最终交付可以是一个 Excel 文件，里面放两个 sheet。
运费险只按子订单编号匹配，这是防止金额偏高的关键规则。
商品成本表和运费险表是可选表，缺失时不要阻塞生成，直接按 0 成本或 0 费用处理。
```
