# 达人实付订单汇总规则

## 一句话结论

以订单表为主表，按达人维度统计实际付款订单表现，并补充由订单号归属回来的快递包裹数、运费险费用和结算到账金额；这是后续按达人计算利润的基础组成部分。

## 适用范围

用于抖音店铺达人结算中，先按达人汇总订单侧的收入、基础成本项和资金流水实际到账金额。

本规则已经用 `风华服饰时尚小铺` 2026 年 5 月订单表、运费险表和资金流水表验证通过。

## 输出字段

最终按达人维度输出 7 个字段：

```text
达人昵称
达人ID
实付订单金额
实付订单数
快递包裹数
运费险费用
结算到账金额
```

## 订单参与条件

只统计 `支付完成时间` 非空的订单参与实付口径：

```text
支付完成时间非空 -> 参与实付订单金额、实付订单数计算
支付完成时间为空 -> 不参与实付订单金额、实付订单数计算
```

原因：

```text
支付完成时间为空，代表用户只是拍下但没有付款，不能作为实际支付 GMV。
```

## 达人归属规则

```text
达人昵称为空 -> 商品卡流量
达人昵称非空 -> 使用原达人昵称
```

如果达人昵称为空，则 `达人ID` 留空。

## 实付订单金额

实付订单金额按订单行计算后，再按达人汇总。

公式：

```text
实付订单金额 =
订单应付金额
+ 平台实际承担优惠金额
+ 商家实际承担优惠金额
+ 达人实际承担优惠金额
```

注意：

```text
只对支付完成时间非空的订单行计算。
```

## 实付订单数

实付订单数统计实际付款的子订单数。

计算方式：

```text
实付订单数 = 支付完成时间非空订单中的去重子订单编号数
```

## 快递包裹数

快递包裹数使用已确认的 [快递包裹数计算规则](./快递包裹数计算规则.md)。

简化口径：

```text
只统计发货时间非空的订单
按 达人昵称/商品卡流量 + 主订单编号 + 发货时间 去重
每个唯一组合算 1 个快递包裹
```

注意：

```text
快递包裹数按发货事实计算，不要求支付完成时间非空。
```

## 运费险费用

运费险费用来自运费险表，但必须根据订单号归属回订单表中的达人。

计算条件：

```text
保费状态 = 已扣减 -> 计入运费险费用
保费状态 = 不扣减 -> 不计入运费险费用
```

费用字段：

```text
运费险费用 = 运费险表中“支付保费”的金额合计
```

归属方式：

```text
1. 读取运费险表的 订单编号
2. 优先匹配订单表的 子订单编号
3. 如果子订单编号匹配不到，再匹配订单表的 主订单编号
4. 匹配成功后，使用订单表里的达人昵称和达人ID归属费用
5. 匹配不到订单表的运费险记录，不归属到任何达人，并作为异常/未匹配数量记录
```

注意：

```text
运费险费用是订单主表达人利润统计的一部分，但来源不是订单表本身，而是通过订单号挂回达人维度的费用项。
```

## 结算到账金额

结算到账金额来自店铺资金流水表，只看资金流水实际发生的入账和出账，不看订单状态。

计算方式：

```text
结算到账金额 = 资金流水入账金额 - 资金流水出账金额
```

方向规则：

```text
动账方向 = 入账 -> 动账金额记为正数
动账方向 = 出账 -> 动账金额记为负数
```

归属方式：

```text
1. 读取资金流水表的 子订单号 和 订单号
2. 优先用 子订单号 匹配订单表的 子订单编号
3. 如果 子订单号 为空或匹配不到，再用 订单号 匹配订单表的 主订单编号
4. 匹配成功后，使用订单表里的达人昵称和达人ID归属到账金额
5. 没有订单号或匹配不到订单表的资金流水，不强行分摊到达人，并作为异常/未归属数量记录
```

注意：

```text
结算到账金额可以为正，也可以为负。
负数是正常情况，代表资金流水中该订单维度存在扣款、退款、服务费等出账。
结算到账金额以资金流水实际入账/出账为准，不用订单状态判断订单是否应结算。
```
